# 职业能力大数据服务平台 — 功能需求测试

> **版本**：v2.0
> **测试类型**：功能测试（黑盒 / 手工）
> **测试环境**：前端 http://localhost:5173 ｜ 后端 http://localhost:8080
> **数据库**：MySQL（多租户共享库）

---

## 一、测试目的

验证系统核心功能是否满足《软件需求规约》中的功能需求（UC）、业务规则（BR）与验收标准（AC）。本轮重点覆盖：**用户登录、职位浏览、职位收藏、职位投递**四个功能点，逐项细化到操作步骤级，并记录实际结果。

---

## 三、测试用例

### 功能点 1：用户登录 `UC-01` / `AC-01`

**功能说明**：用户输入租户、账号、密码，系统校验后签发 JWT Token，按角色跳转对应首页。

| 用例 | 前置条件 | 操作步骤 | 预期结果 | 实际结果 | 通过 |
|---|---|---|---|---|---|
| TC-01-1 正常登录 | 账号存在且启用 | 1. 打开 /login<br>2. 选择"测试学院"<br>3. 输入 student01 / student123<br>4. 点击登录 | 提示"欢迎回来"，跳转 /student，顶栏显示姓名与角色 | | ☐通过 |
| TC-01-2 密码错误 | 同上 | 输入 student01 / wrongpwd → 登录 | 提示"用户名或密码错误"，停留登录页 | | ☐通过 |
| TC-01-3 账号不存在 | — | 输入 nobody / 任意密码 → 登录 | 提示登录失败，不跳转 | | ☐通过 |
|                          |                |                                                              |                                                   |          |       |
| TC-01-5 空用户名 | — | 租户已选，用户名留空 → 登录 | 提示"请输入用户名" | | ☐通过 |
| TC-01-6 空密码 | — | 用户名已填，密码留空 → 登录 | 提示"请输入密码" | | ☐通过 |
| TC-01-7 已登录访问登录页 | 已登录 | 地址栏手动输入 /login | 自动跳回角色首页，不显示登录框 | | ☐通过 |
| TC-01-8 未登录访问内页 | 未登录 | 地址栏直接输入 /student | 自动重定向到 /login | | ☐通过 |

**接口**：`POST /api/auth/login`、`GET /api/auth/tenants`

---

### 功能点 2：职位浏览（推荐 + 详情）`UC-04` `UC-07` / `AC-06`

**功能说明**：学生进入推荐页看到按匹配分排序的职位卡片；点击卡片进入详情页查看完整信息。

| 用例 | 前置条件 | 操作步骤 | 预期结果 | 实际结果 | 通过 |
|---|---|---|---|---|---|
| TC-02-1 推荐列表加载 | 学生已登录且有画像 | 进入 /student | 显示推荐职位卡片（最多 20 条），每张含职位名、公司、城市、薪资、学历 | | ☐通过 |
| TC-02-2 匹配分展示 | 同上 | 查看任一卡片右上角 | 显示"匹配分"数值，列表按分值从高到低排序 | | ☐通过 |
| TC-02-3 匹配原因 | 同上 | 查看卡片正文 | 显示匹配原因文字（如技能/城市契合） | | ☐通过 |
| TC-02-4 缺失技能建议 | 同上 | 查看卡片"建议学习"区 | 列出该职位要求但画像缺失的技能 chip；无缺失则不显示该区 | | ☐无 |
| TC-02-5 无画像引导 | 学生未填画像 | 进入 /student | 显示空状态"暂无推荐职位，请先完善个人画像" | | ☐无 |
| TC-02-6 进入详情 | 推荐列表已加载 | 点击任一职位卡片 | 跳转 /student/job/:id，页头显示职位名 | | ☐通过 |
| TC-02-7 详情内容完整 | 在详情页 | 查看详情卡片 | 显示城市/行业/学历/经验/薪资/发布日期、技能要求 chip、职位描述 | | ☐通过 |
| TC-02-8 职位不存在 | — | 手动访问 /student/job/999999 | 显示空状态"职位不存在" | | ☐通过 |
| TC-02-9 返回上一页 | 在详情页 | 点击页头返回箭头 | 回到上一页面（推荐/收藏列表） | | ☐通过 |

**接口**：`GET /api/student/recommend`、`GET /api/job/{id}`

---

### 功能点 3：职位收藏 / 取消收藏 `UC-08` / `AC-07`

**功能说明**：学生在详情页收藏职位，收藏记录写入行为表；收藏后在"我的收藏"页可见；可取消收藏。

| 用例 | 前置条件 | 操作步骤 | 预期结果 | 实际结果 | 通过 |
|---|---|---|---|---|---|
| TC-03-1 收藏职位 | 详情页，未收藏 | 点击"收藏职位"按钮 | 提示"已收藏"，按钮变为"取消收藏"（高亮） | | ☐通过 |
| TC-03-2 收藏状态回显 | 已收藏该职位 | 刷新详情页 / 重新进入 | 按钮初始即为"取消收藏"，状态正确回显 | | ☐通过 |
| TC-03-3 收藏页展示 | 已收藏 ≥1 个职位 | 进入 /student/favorites | 列表展示已收藏职位卡片（职位名/公司/城市/薪资） | | ☐通过 |
| TC-03-4 从收藏进详情 | 收藏页有数据 | 点击收藏卡片 | 跳转对应职位详情页 | | ☐通过 |
| TC-03-5 取消收藏 | 详情页，已收藏 | 点击"取消收藏" | 提示"已取消收藏"，按钮变回"收藏职位" | | ☐通过 |
| TC-03-6 取消后列表更新 | 刚取消收藏 | 进入 /student/favorites | 该职位不再出现在收藏列表 | | ☐通过 |
| TC-03-7 空收藏引导 | 无任何收藏 | 进入 /student/favorites | 显示空状态"暂无收藏，去推荐页看看吧" | | ☐通过 |

**接口**：`POST /api/student/favorite/{jobId}`、`DELETE /api/student/favorite/{jobId}`、`GET /api/student/favorites`
**数据库验证**：`student_behavior` 表新增 `behavior_type='FAVORITE'` 记录；取消后该记录被删除或标记失效。
