# 部署与同步说明

本文档用于保证团队成员从 GitHub 拉取代码后，能用同一套步骤初始化数据、启动服务，并部署到云服务器。

## 本地初始化数据

开发环境数据库由 `occupation-common/src/main/resources/sql/init.sql` 初始化。该文件由脚本生成，不要手工改大段 INSERT 数据。

推荐步骤：

```powershell
.\scripts\reset-dev-db.ps1
```

脚本会自动执行：

1. 重新生成 `mock-jobs.json` 和 `init.sql`。
2. 重建 Docker 开发环境数据库卷。
3. 启动 MySQL、Redis、Zookeeper、Kafka。
4. 等待 MySQL 完成健康检查。

如果只想使用当前已生成的 SQL，不重新生成数据：

```powershell
.\scripts\reset-dev-db.ps1 -SkipBuildData
```

注意：该脚本会删除本地 Docker 开发数据库卷，适合开发/演示环境，不要在生产服务器上执行。

## 当前种子数据规模

当前初始化数据控制在中等规模，避免页面分析过空，也避免初始化和查询变慢：

- Mock 采集源：120 条职位。
- 初始化职位：约 180 个职位，包含采集职位与 24 个 HR 站内职位。
- 投递数据：每个 HR 岗位都有候选人，覆盖待查看、已查看、面试、录用、拒绝等状态。
- 行为数据：包含浏览、收藏、投递、自主联系、忽略，用于学生、教师、HR、管理员工具箱分析。

## GitHub 同步约定

提交代码前请确认这些文件一起提交：

- `scripts/gen-seed-data.js`
- `scripts/gen-mock-jobs.js`
- `occupation-common/src/main/resources/sql/init.sql`
- `occupation-crawler/src/main/resources/mock/mock-jobs.json`
- `memory-bank/architecture.md`

不要提交：

- `target/`
- `dist/`
- `node_modules/`
- `.env`
- `application-local.yml`
- `logs/`
- `data/reports/`

## 云服务器部署

服务器需要安装 Docker 和 Docker Compose。

1. 拉取代码。

```bash
git pull
```

2. 配置环境变量。

```bash
cp .env.example .env
```

至少修改：

- `DB_PASSWORD`
- `REDIS_PASSWORD`
- `JWT_SECRET`

3. 启动生产环境。

```bash
docker compose -f docker-compose.prod.yml --env-file .env up -d --build
```

生产 compose 会自动构建后端 JAR 和前端静态资源，不依赖本地已有的 `target` 或 `dist`。

4. 查看状态。

```bash
docker compose -f docker-compose.prod.yml ps
docker logs occupation-app
```

首次启动时 MySQL 会从 `occupation-common/src/main/resources/sql/init.sql` 初始化数据。若生产库已经存在数据，Docker 不会重复执行 init.sql。

## 外部资讯源

管理端“拉取外部资讯”默认使用开源中国 RSS，不再默认 Google News。原因是国内服务器和多数云环境访问 Google 不稳定，容易出现“拉取 0 条”的误判。

当前策略：

- 先按关键词过滤 RSS 标题/摘要。
- 如果关键词没有命中，则回退拉取最新资讯，避免用户误以为功能坏了。
- 已存在同标题资讯会自动跳过，避免重复入库。
## 数据采集与行业资讯边界

这两个功能不要混用：

- 管理员端“采集任务管理”只负责岗位/公告类数据，结果进入 raw_job_data、job_detail 和分析面板，用于行业热度、城市分布、技能需求等统计。
- 管理员端“资讯管理”负责行业资讯/新闻，结果进入 news 表，用于学生端、教师端、HR 端、管理员端的行业资讯页面和首页资讯卡片。

如果要补充新闻资讯，请进入“资讯管理”点击“拉取外部资讯”；如果要补充数据面板样本，请使用 MOCK 或 `OFFICIAL_PUBLIC` 采集任务。

如果要采集公开招聘公告并进入岗位分析链路，请使用 `OFFICIAL_PUBLIC`。招聘网站来源如 `ZHAOPIN` 保留兼容，但不再作为默认初始化任务，因为真实招聘站点经常受 robots、反爬、页面改版和网络环境影响，容易出现 0 条或任务失败。
