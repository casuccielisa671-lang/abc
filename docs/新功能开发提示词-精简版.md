## 1.技能投资回报率（ROI）分析

### 功能描述
学生输入技能名称（如 "Docker"），系统对比「要求该技能」与「不要求该技能」的岗位数据，量化学习该技能的薪资回报与市场需求，帮助学生判断"值不值得学"。

### 核心逻辑
1. 从 `job_detail` 表中解析技能标签字段，区分"要求该技能"和"不要求该技能"两组岗位
2. 多维度对比统计：
   - **需求规模**：要求该技能的岗位数量 vs 不要求的岗位数量（计算需求占比）
   - **薪资溢价**：两组岗位的平均薪资、中位数薪资、P25/P75 薪资范围
   - **需求趋势**：近 6~12 个月要求该技能的岗位数量变化（按月），判断需求是上升还是下降
   - **城市分布**：要求该技能的岗位在哪些城市更集中（Top 5 城市）
   - **关联技能**：与该技能常同时出现的 Top 5 技能（技能共现分析）
3. 计算核心指标：
   - `薪资溢价率 = (有技能平均薪资 - 无技能平均薪资) / 无技能平均薪资 × 100%`
   - `需求占比 = 有技能岗位数 / 总岗位数 × 100%`
   - `趋势方向`：上升 / 平稳 / 下降（基于近 6 个月斜率）
4. 综合评分（0~100 分）：加权计算薪资溢价、需求占比、趋势方向的综合得分
5. 生成自然语言评估结论 + 学习建议

## 3.期望薪资计算器

### 功能描述
用户输入城市 + 岗位 + 学历 + 经验年限，系统基于历史招聘数据给出合理期望薪资范围，并展示同条件下薪资分布和市场竞争力分析。

### 核心逻辑
1. 从 `job_detail` 表中按多条件筛选匹配岗位（城市、岗位名称模糊匹配、学历、经验年限范围）
2. 数据量不足时自动放宽条件（如去掉城市限制、放宽经验年限 ±1 年），并标注"数据样本较少，结果仅供参考"
3. 统计薪资分布：
   - P10 / P25 / P50（中位数）/ P75 / P90
   - 均值 ± 标准差
   - 薪资分布直方图数据（分桶统计）
4. 多维对比：
   - 同城市同岗位 vs 全国同岗位（城市溢价）
   - 同经验年限 vs 相邻经验段（经验溢价）
   - 同学历 vs 相邻学历段（学历溢价）
5. 输出建议：期望薪资范围 `[P25, P75]`，面试报价参考 P50，底线参考 P25

## 4.多岗位对比工具

### 功能描述
选择 2~4 个岗位，从薪资、技能要求、学历门槛、城市分布、需求趋势、竞争热度等多个维度并排对比，帮助学生在多个职业方向间做出选择。

### 核心逻辑
1. 用户搜索并选择 2~4 个岗位（限制上限防止信息过载）
2. 系统查询每个岗位的多维度数据：
   - **薪资维度**：平均薪资、中位数、P25/P75 范围、近 6 个月薪资趋势
   - **技能维度**：Top 10 高频技能要求（含出现频率百分比）
   - **学历维度**：学历要求分布（大专/本科/硕士/博士/不限）
   - **城市维度**：岗位城市分布 Top 5（含数量和占比）
   - **趋势维度**：近 12 个月招聘量变化趋势
   - **竞争维度**：竞争热度指数（投递数/岗位数）、供需比
3. 对比分析输出：
   - **薪资对比**：最高/最低薪资岗位标注，差值高亮
   - **技能交集**：所有岗位共同要求的技能（通用技能）
   - **技能差异**：每个岗位独有的技能要求
   - **综合推荐**：基于薪资、趋势、竞争度加权打分，排序推荐

## 5.求职清单生成器

### 功能描述
学生选择目标岗位后，系统自动对比岗位要求与个人技能库，生成「技能差距分析 + 优先级排序 + 学习路径建议 + 推荐资源」的完整求职准备清单，帮助学生明确"还差什么、先学什么、怎么学"。

### 核心逻辑
1. 从 `job_detail` 中提取目标岗位的技能要求，按出现频率排序（高频 = 必备，低频 = 加分项）
2. 读取学生当前技能列表（`sys_student_profile.skills`），做差集计算
3. 缺失技能分级：
   - **HIGH（必备）**：岗位中出现频率 > 70% 的技能，不掌握基本无法应聘
   - **MEDIUM（重要）**：频率 40%~70%，掌握后有竞争力
   - **LOW（加分）**：频率 < 40%，锦上添花
4. 为每个缺失技能生成：
   - 预估学习周期（入门级/熟练级）
   - 推荐学习资源（优先匹配系统预配置资源库，无匹配时用规则生成搜索链接）
   - 前置依赖提示（如学 Spring Boot 前需先掌握 Java）
5. 生成学习路线图：按优先级 + 前置依赖排序，给出推荐学习顺序
6. 可选：调用 LLM 生成个性化学习建议（降级为规则模板）

## 7.岗位衰退预警

### 功能描述
监控岗位连续 3 个月招聘量下降，自动标记为"衰退岗位"，提醒学生谨慎选择。

### 核心逻辑
1. 按岗位类别（`job_detail.category` 或 `job_detail.title` 关键词聚类）统计月度招聘量
2. 检测连续 3 个月招聘量环比下降的岗位
3. 计算下降幅度：`(当前月 - 3月前) / 3月前 × 100%`
4. 生成预警列表 + 推送通知

### 接口设计
```
GET /api/analysis/declining-jobs?threshold=3&minDecline=15
Response: {
  "decliningJobs": [
    {
      "category": "PHP开发",
      "currentCount": 200,
      "threeMonthsAgoCount": 350,
      "declinePercent": 42.8,
      "monthlyTrend": [
        { "month": "2026-04", "count": 350 },
        { "month": "2026-05", "count": 280 },
        { "month": "2026-06", "count": 220 },
        { "month": "2026-07", "count": 200 }
      ],
      "warningLevel": "HIGH",
      "suggestion": "PHP开发岗位连续3个月下降42.8%，建议关注相关替代技能（如Go、Node.js）"
    }
  ]
}
```

### 技术要点
- 后端：`occupation-analysis` 模块新增分析维度
- 定时任务（`@Scheduled`）每月/每周自动计算
- 结果写入 `analysis_result` 表（dimension=job_decline）
- 预警通知：写入 `sys_alert` 表 + 前端轮询/WebSocket

### 前端
- 学生端 + 教师端展示预警列表
- 红色/黄色标记衰退等级
- 可点击查看趋势折线图

## 8，.岗位-技能二维矩阵热力图

### 功能描述
行 = 岗位类别，列 = 技能名称，颜色深浅 = 该技能在该岗位中被要求的频率。一眼看出"哪些技能是通用硬通货"。

### 核心逻辑
1. 从 `job_detail` 表中提取岗位类别 + 技能标签
2. 构建共现矩阵：`matrix[岗位][技能] = 要求该技能的该岗位数量`
3. 归一化：按行（岗位）归一化，使热力图颜色反映"该岗位中该技能的相对重要性"
4. 可选：按列排序，把"通用技能"排在前面

### 接口设计
```
GET /api/analysis/skill-job-matrix?topN=20
Response: {
  "jobs": ["Java开发", "前端开发", "数据分析", "AI工程师", ...],
  "skills": ["Java", "Python", "SQL", "Spring", "Docker", ...],
  "matrix": [
    [1.0, 0.2, 0.8, 0.9, 0.5, ...],  // Java开发 行
    [0.1, 0.3, 0.7, 0.1, 0.2, ...],  // 前端开发 行
    ...
  ],
  "universalSkills": ["SQL", "Python", "Linux"]  // 出现频率最高的技能
}
```

### 技术要点
- 后端：`occupation-analysis` 模块新增计算逻辑
- 数据量大时可预计算 + 缓存（Redis）
- 前端：使用 ECharts Heatmap 或自定义 Canvas 热力图
- 支持点击单元格查看具体岗位列表

### 前端
- 新增页面：`/analysis/skill-matrix`
- ECharts 热力图组件
- 支持筛选岗位类别、技能类别

## 开发注意事项

1. **数据层**：数据库已就绪，关键表结构如下：
   - `job_detail`：title, company, city, industry, salary_min/max, education, experience, skills(TEXT/JSON), description, publish_date, source, publisher_id
   - `job_application`：user_id, job_id, publisher_id, status(SUBMITTED/VIEWED/INTERVIEW/OFFER/REJECTED), hr_note
   - `student_behavior`：user_id, job_id, action(VIEW/FAVORITE/APPLY/IGNORE/CONTACT)
   - `sys_student_profile`：major, skills(TEXT), expected_city, expected_salary_min/max, education_level
   - `student_resume`：educations(JSON), projects(JSON), internships(JSON), honors(JSON)
   - `analysis_result`：dimension, dimension_value, metric_name, metric_value, period_type, period_value
   - 技能标签存储格式：`job_detail.skills` 为 TEXT 类型，实际存储 JSON 数组（如 `["Java","Spring","MySQL"]`）
   - 城市字段无枚举约束，自由文本（如"北京"、"上海"）
   - 学历枚举：专科/本科/硕士/博士/不限
2. **可视化库**：开发可视化功能时，根据图表类型选择最合适的库（ECharts 覆盖面广，D3.js 更灵活）
3. **求职清单生成器**：保留开发，优先级与其他功能相同
4. **AI 功能**：ROI 结论生成和衰退预警建议先用规则模板实现，后续可升级为 LLM 生成
5. **工具箱后端开发**：12 个子工具的后端 API 均需新建，建议按角色分批开发：
   - 第一批（学生端 4 个）：技能 ROI、期望薪资计算器、多岗位对比、求职清单生成器
   - 第二批（教师端 3 个）：班级就业对比、学生预警看板、课程-岗位匹配
   - 第三批（HR 端 3 个）：JD 优化助手、人才对比、薪资竞争力分析
   - 第四批（管理员 2 个）：数据导出中心、租户健康度监控
