## 一、教师端工具

### 1. 班级就业对比

#### 功能描述
教师选择多个班级，从就业率、平均薪资、去向分布、技能掌握率等维度并排对比，辅助教学评估与课程改革决策。

#### 核心逻辑
1. 教师选择 2~5 个班级（同租户下）
2. 系统统计每个班级的就业数据：
   - **就业率**：已投递/已入职学生数 / 班级总人数
   - **平均薪资**：班级学生画像中期望薪资的均值/中位数
   - **去向分布**：学生投递的岗位类别 Top 5（含人数和占比）
   - **技能掌握率**：班级学生画像中高频技能 Top 10（含掌握人数和占比）
   - **画像完整度**：班级学生画像填写率
3. 对比输出：
   - 差异高亮：就业率最高/最低班级标注
   - 共同去向：所有班级学生都倾向的岗位方向
   - 薄弱技能：某班级普遍缺失但市场需求的技能

#### 接口设计
```
POST /api/teacher/compare-classes
Request: { "classIds": [1, 2, 3] }
Response: {
  "classes": [
    {
      "id": 1, "name": "2023级软件工程1班",
      "studentCount": 35,
      "employmentRate": 82.9,
      "avgSalary": 15000,
      "medianSalary": 14000,
      "topDestinations": [
        { "jobCategory": "Java开发", "count": 12, "ratio": 34.3 },
        { "jobCategory": "前端开发", "count": 8, "ratio": 22.9 }
      ],
      "topSkills": [
        { "name": "Java", "masteredCount": 28, "ratio": 80.0 },
        { "name": "Python", "masteredCount": 25, "ratio": 71.4 }
      ],
      "profileCompletionRate": 88.6
    }
  ],
  "comparison": {
    "highestEmployment": "2023级软件工程1班",
    "lowestEmployment": "2023级软件工程2班",
    "commonDestinations": ["Java开发", "前端开发"],
    "weakSkills": {
      "2023级软件工程2班": ["Docker", "Redis"]
    }
  }
}
```

#### 前端
- 教师端新增页面：`/teacher/tools/compare-classes`
- 组件拆分：
  - `ClassMultiSelect.vue`：班级多选组件（最多 5 个）
  - `EmploymentCompareChart.vue`：就业率/薪资对比柱状图
  - `DestinationCompare.vue`：去向分布对比（分组柱状图）
  - `SkillRadarCompare.vue`：技能雷达图（多班级叠加）
  - `ComparisonSummary.vue`：对比总结卡片（差异高亮）

### 2. 学生预警看板

#### 功能描述
自动标记画像不完整、长期未投递、技能缺口大的学生，帮助教师精准定位需要辅导的学生。

#### 核心逻辑
1. 按预警类型分类展示学生列表：
   - **画像不完整**：`sys_student_profile` 中 skills 为空或期望薪资未填
   - **长期未投递**：近 30 天无 `job_application` 记录
   - **技能缺口大**：学生技能与热门岗位要求匹配度 < 30%
   - **投递无反馈**：投递后超过 14 天状态仍为 SUBMITTED
2. 每类预警展示学生列表 + 关键指标
3. 支持按班级筛选、按预警等级排序
4. 点击学生可查看详细画像

#### 接口设计
```
GET /api/teacher/student-alerts?classId=1&alertType=ALL
Response: {

    {
      "userId": 101,
      "realName": "张三",

      "alertType": "SKILL_GAP",
      "alertLevel": "HIGH",

}
```

#### 前端
- 教师端新增页面：`/teacher/tools/student-alerts`
- 组件拆分：
  - `AlertSummaryCards.vue`：四类预警数量概览卡片
  - `AlertTypeTabs.vue`：按预警类型切换的 Tab
  - `StudentAlertTable.vue`：预警学生列表表格（支持排序、筛选）
  - `StudentDetailDrawer.vue`：学生详情抽屉（画像 + 投递记录）

### 3. 课程-岗位匹配分析

#### 功能描述
教师输入课程名称，查看该课程关联的市场岗位需求趋势，辅助判断课程内容是否与市场接轨。

#### 核心逻辑
1. 教师输入课程名称（如"Java程序设计"）
2. 系统通过关键词映射（课程名 → 技能标签 → 岗位类别），找到关联岗位
3. 展示关联岗位的：
   - 招聘量趋势（近 12 个月）
   - 平均薪资走势
   - 技能要求变化（新增/淘汰的技能）
4. 输出课程建议：哪些技能需要加入教学大纲、哪些可以弱化

#### 接口设计
```
POST /api/teacher/course-market-match
Request: { "courseName": "Java程序设计" }
Response: {
  "courseName": "Java程序设计",
  "mappedSkills": ["Java", "OOP", "Spring", "MySQL"],
  "relatedJobs": [
    { "title": "Java开发", "relevance": 0.95, "avgSalary": 20000, "demandTrend": "上升" },
    { "title": "后端开发", "relevance": 0.85, "avgSalary": 19000, "demandTrend": "平稳" }
  ],
  "skillChanges": {
    "emerging": ["Spring Cloud", "Kubernetes"],
    "declining": ["JSP", "Struts"]
  },
  "suggestion": "建议在课程中增加 Spring Boot 实战和 Docker 基础内容，弱化 JSP 部分"
}
```

#### 前端
- 教师端新增页面：`/teacher/tools/course-match`
- 组件拆分：
  - `CourseSearchInput.vue`：课程名称搜索输入框
  - `RelatedJobsList.vue`：关联岗位列表（含相关度、薪资、趋势标签）
  - `SkillChangeTimeline.vue`：技能变化时间轴（新兴 vs 衰退）
  - `SuggestionCard.vue`：课程建议卡片

## 二、管理员端工具

### 4. 数据导出中心

#### 功能描述
管理员可批量导出数据看板、就业分析、用户数据等为 Excel/PDF，方便汇报和存档。

#### 核心逻辑
1. 导出类型选择：
   - **数据看板**：Dashboard 5 维度数据
   - **就业分析**：就业率、投递漏斗、供需错配数据
   - **用户数据**：用户列表（按角色筛选）
   - **采集日志**：爬虫任务执行记录
2. 支持选择时间范围（本月/本季度/自定义）
3. 导出格式：Excel（.xlsx）或 PDF
4. 大文件异步导出 + 下载链接通知

#### 接口设计
```
POST /api/admin/export
Request: {
  "type": "DASHBOARD",
  "format": "EXCEL",
  "dateRange": { "start": "2026-01-01", "end": "2026-06-30" }
}
Response: { "taskId": "exp_20260713_001", "status": "PROCESSING" }

GET /api/admin/export/{taskId}/status
Response: { "taskId": "exp_20260713_001", "status": "COMPLETED", "downloadUrl": "/api/admin/export/download/exp_20260713_001" }
```

#### 前端
- 管理员端新增页面：`/admin/tools/export`
- 组件拆分：
  - `ExportTypeSelector.vue`：导出类型选择（卡片式）
  - `ExportForm.vue`：时间范围 + 格式选择表单
  - `ExportTaskList.vue`：导出任务列表（状态 + 下载按钮）

### 5. 租户健康度监控

#### 功能描述
展示各租户的活跃度、数据完整度、API 调用量等运营指标，帮助管理员掌握平台整体运行状况。

#### 核心逻辑
1. 按租户维度统计：
   - **活跃度**：近 30 天登录用户数、活跃用户占比
   - **数据完整度**：岗位数据量、学生画像填写率、投递记录数
   - **功能使用率**：各功能模块的访问次数
   - **API 调用量**：近 30 天 API 请求数（如有记录）
2. 健康度评分（0~100）：加权计算各指标
3. 异常标记：数据量骤降、活跃度异常等

#### 接口设计

## 三、HR 端工具

### 6. JD 优化助手

#### 功能描述
AI 分析职位描述质量，从完整性、吸引力、关键词覆盖等维度给出优化建议，帮助 HR 写出更高效的 JD。

#### 核心逻辑
1. HR 输入/粘贴 JD 内容（或选择已有职位）
2. 系统分析：
   - **完整性检查**：是否包含岗位职责、任职要求、薪资范围、福利待遇
   - **关键词覆盖**：与同类热门 JD 对比，缺失的高频关键词
   - **吸引力评分**：语言风格、结构清晰度、亮点突出度
   - **薪资竞争力**：该岗位薪资在市场中的分位
3. 输出优化建议 + 优化后的 JD 草稿

#### 接口设计
```
POST /api/hr/jd-optimize
Request: {
  "jobId": 5,  // 可选，已有职位
  "content": "岗位职责：..."  // 可选，手动输入
}
Response: {
  "score": 72,
  "checks": {
    "completeness": { "passed": true, "missing": [] },
    "keywords": { "passed": false, "missing": ["五险一金", "弹性工作", "年终奖"], "suggestion": "建议补充福利待遇关键词" },
    "attractiveness": { "score": 68, "suggestion": "岗位亮点不够突出，建议前置核心优势" },
    "salaryCompetitiveness": { "percentile": 60, "suggestion": "薪资处于市场中上水平，可在 JD 中强调" }
  },
  "optimizedDraft": "【岗位亮点】...\n\n岗位职责：\n1. ...",
  "improvementTips": ["增加具体福利描述", "前置岗位亮点", "补充团队介绍"]
}
```

#### 前端
- HR 端新增页面：`/hr/tools/jd-optimizer`
- 组件拆分：
  - `JdInputArea.vue`：JD 内容输入区（支持粘贴 + 选择已有职位）
  - `ScoreOverview.vue`：综合评分环形图
  - `CheckItemList.vue`：各项检查结果列表（通过/未通过 + 建议）
  - `OptimizedPreview.vue`：优化后 JD 预览（左右对比或高亮差异）

### 7. 人才对比

#### 功能描述
选择 2~4 个候选人，从技能、学历、经验、期望薪资等维度并排对比，辅助筛选决策。

#### 核心逻辑
1. HR 从人才浏览或投递列表中勾选候选人
2. 对比维度：
   - **基本信息**：学历、专业、毕业院校
   - **技能匹配**：候选人技能 vs 岗位要求技能（匹配度百分比）
   - **经验**：工作年限、项目经验关键词
   - **期望薪资**：与岗位预算的匹配度
   - **投递状态**：投递时间、当前状态
3. 差异高亮 + 综合推荐排序

#### 接口设计
```
POST /api/hr/compare-candidates
Request: { "userIds": [101, 102, 103], "jobId": 5 }

      "experienceYears": 3,
      "expectedSalary": 18000,
      "applicationStatus": "VIEWED",
      "appliedAt": "2026-07-10"

    "commonSkills": ["Java", "MySQL"],
    "uniqueSkills": {
      "101": ["Redis"],

    }
  }
}
```

#### 前端
- HR 端新增页面：`/hr/tools/compare-candidates`

  - `CandidateMultiSelect.vue`：候选人搜索多选组件

  - `SkillMatchRadar.vue`：技能匹配雷达图（多候选人叠加）

### 8. 薪资竞争力分析

#### 功能描述
HR 输入岗位和薪资范围，对比市场数据，判断自家 offer 是否有竞争力。

#### 核心逻辑
1. HR 输入：岗位名称 + 城市 + 薪资范围
2. 系统查询该岗位在该城市的市场薪资分布
3. 计算自家薪资在市场中的分位（P10~P90）
4. 输出竞争力评估：
   - **低于 P25**：竞争力弱，建议提薪
   - **P25~P50**：竞争力一般
   - **P50~P75**：竞争力良好
   - **高于 P75**：竞争力强

#### 接口设计
```
POST /api/hr/salary-competitiveness
Request: {
  "jobTitle": "Java开发",
  "city": "北京",
  "salaryRange": [15000, 22000]
}
Response: {
  "marketData": {
    "p10": 10000, "p25": 14000, "p50": 19000, "p75": 26000, "p90": 35000,
    "avg": 21000, "sampleSize": 280
  },
  "yourPosition": {
    "midpoint": 18500,
    "percentile": 48,
    "level": "一般",
    "suggestion": "您的薪资中位点处于市场 P48，竞争力一般。建议将薪资范围调整为 [17000, 26000] 以覆盖 P50~P75 区间。"
  },
  "distribution": [
    { "range": "8k-12k", "count": 28, "ratio": 10.0 },
    { "range": "12k-16k", "count": 56, "ratio": 20.0 }
  ]
}
```

#### 前端
- HR 端新增页面：`/hr/tools/salary-compete`
- 组件拆分：
  - `SalaryInputForm.vue`：岗位 + 城市 + 薪资范围输入表单
  - `CompetitivenessGauge.vue`：竞争力仪表盘（分位指针）
  - `DistributionChart.vue`：市场薪资分布直方图（标注自家位置）
  - `SuggestionCard.vue`：调整建议卡片

## 五、开发决策记录

| 问题 | 决策 |
|------|------|
| 开发顺序 | 先写前端界面（静态/Mock 数据），后端 API 后续对接 |
| 工具箱入口 | 每端各一个 Toolbox.vue 聚合页，风格与学生端一致（简约卡片式） |
| 后端模块归属 | 教师端工具 → `occupation-recommend`；管理员工具 → `occupation-web`；HR 工具 → `occupation-recommend` |
| 数据依赖 | 班级对比、学生预警、人才对比等需要 `sys_student_profile`、`job_application` 等表数据，前端先用 Mock 数据开发 |
| AI 功能 | JD 优化助手可后续接入 LLM，当前用规则模板 |
| 图表库 | 统一使用 ECharts |
