## 2. 项目目标与范围

### 2.1 项目目标

构建面向高校、职业院校、学生、教师及企业 HR 的职业能力大数据服务平台，实现：

1. **全量数据采集**：自动化、分布式采集主流招聘平台与官方公开就业数据。
2. **多维分析洞察**：对岗位、行业、技能、地域、学历等维度进行统计分析。
3. **智能报告生成**：基于模板自动生成 PDF / Word / HTML 就业分析报告。
4. **精准岗位推荐**：结合学生画像与职位标签进行匹配推荐。
5. **标准化开放 API**：向第三方系统提供受 OAuth2 保护的 REST API。

### 2.2 项目范围

**包含范围**：

- 管理后台（管理员、教师、HR 端）
- 学生端（Web）
- 数据采集与调度系统
- 大数据分析引擎
- 报告生成与导出系统
- 岗位推荐与推送系统
- 对外公开 API 与文档
- 统一认证、鉴权、多租户隔离
- 部署脚本与开发/测试环境配置

**不包含范围**：

- 移动端 App 原生客户端（P5 后可选扩展）
- 生产环境 K8s 集群自动扩缩容（保留扩展能力）
- 与校方教务系统、HR 系统的深度对接（仅预留 API 接口）
- 商业化运营与计费系统

## 7. 风险管理

| 风险 | 影响 | 可能性 | 应对策略 |
|---|---|---|---|
| 招聘平台反爬升级 | 数据采集失败 | 中 | 设计代理池、UA 池、降级为离线测试数据 |
| 推荐算法效果不达预期 | 岗位匹配质量差 | 中 | 先以规则引擎兜底，后续引入机器学习模型 |
| 大数据组件资源不足 | 本地无法运行完整大数据链路 | 高 | 开发期先用 MySQL 模拟，测试环境补 Spark/Hive |
| 团队并行开发冲突 | 接口契约变更导致 B/C 编译失败 | 高 | 先稳定 A 组接口，每日同步契约变更 |
| 第三方依赖授权问题 | 字体/类库商用受限 | 低 | 优先使用开源可商用组件（Apache 2.0 / MIT） |

## 9. 验收标准

项目整体验收需满足：

1. 全部计划功能完成，并通过单元测试与集成测试。
2. 核心业务流程（采集→分析→报告→推荐→推送）端到端跑通。
3. 管理后台、学生端、教师端、HR 端可正常访问。
4. 对外 API 文档可访问，关键接口可被测试客户端调用。
5. 系统运行稳定，无 P0/P1 级别缺陷。
6. 文档齐全：架构文档、数据库文档、接口文档、部署手册、用户手册。
