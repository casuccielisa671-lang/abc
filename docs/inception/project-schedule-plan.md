## 7. 关键路径

```
项目启动 → 需求确认 → 架构设计 → 数据库设计 → 接口定义 → 环境搭建 → 认证与多租户 → 数据分析 → 报告生成 → 推荐匹配 → 前端联调 → 集成测试 → 系统测试 → UAT → 上线
```

**关键路径说明**：
- 数据分析（IP-3.4）是报告生成（IP-3.6）和职位查询（IP-3.5）的前置。
- 职位查询（IP-3.5）是推荐匹配（IP-3.8）的前置。
- 所有后端接口完成后，D 组前端才能联调（IP-3.10）。
- 集成测试（IP-3.11）是验收阶段入口。

## 8. 资源负荷与并行策略

### 8.1 并行策略

- A 组与 B 组在精化阶段需同步确认接口契约，构建阶段可并行。
- C 组在 A 组职位查询接口就绪后即可开始开发，与 B 组并行。
- D 组前端在精化阶段完成原型，构建阶段中后期与后端并行联调。
- 测试从构建阶段中期开始介入，编写并执行测试用例。

### 8.2 缓冲时间

- 每个阶段预留 0.5~1 天用于评审返工。
- 构建阶段预留 1 天用于集成测试与缺陷修复。
- 验收阶段预留 2 天用于 UAT 问题修复与回归测试。

## 9. 跟踪与报告机制

### 9.1 跟踪频率

- 每日站会更新任务状态。
- 每周五更新项目进度甘特图。
- 阶段末召开里程碑评审会。

### 9.2 进度偏差处理

| 偏差范围 | 处理措施 |
|---|---|
| ≤ 1 天 | 由组长自行调整任务优先级，加班追赶 |
| 1~3 天 | 项目经理协调资源、砍低优先级功能、调整后续计划 |
| > 3 天 | 上报项目经理，启动变更控制流程 |
