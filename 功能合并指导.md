# AI 功能合并指导

> 本文档列出项目中所有 AI 功能的实现位置，供队友合并到自己的代码分支。

## 前置依赖（必须先合并）

### 1. `occupation-common` 模块 — AI 基础设施

| 文件 | 路径 | 说明 |
|------|------|------|
| `AiChatClient.java` | `occupation-common/src/main/java/com/occupation/common/ai/AiChatClient.java` | **核心**：OpenAI 兼容 `/chat/completions` 客户端，全平台唯一 LLM 出口 |
| `AiProperties.java` | `occupation-common/src/main/java/com/occupation/common/ai/AiProperties.java` | 配置类，对应 `app.ai.*` |
| `AiMessage.java` | `occupation-common/src/main/java/com/occupation/common/ai/AiMessage.java` | 消息实体（system/user/assistant） |
| `AiUnavailableException.java` | `occupation-common/src/main/java/com/occupation/common/ai/AiUnavailableException.java` | AI 不可用异常 |

### 2. 配置文件

**`occupation-web/src/main/resources/application.yml`** — 添加 AI 配置块：

```yaml
app:
  ai:
    enabled: ${AI_ENABLED:false}
    base-url: ${AI_BASE_URL:https://api.deepseek.com/v1}
    api-key: ${AI_API_KEY:}
    model: ${AI_MODEL:deepseek-chat}
    temperature: 0.3
    max-tokens: 1200
    connect-timeout-ms: 5000
    read-timeout-ms: 60000
    max-retries: 1
```

**`occupation-web/src/main/resources/application-local.yml`** — 本地密钥（已 gitignore）：

```yaml
app:
  ai:
    enabled: true
    api-key: sk-你的密钥
    base-url: https://api.deepseek.com/v1
    model: deepseek-chat
```

### 功能 2：AI 求职分析 & 匹配分数

#### 后端

| 层级 | 文件 | 路径 |
|------|------|------|
| Service 接口 | `SemanticMatchService.java` | `occupation-recommend/.../service/SemanticMatchService.java` |
| Service 实现 | `SemanticMatchServiceImpl.java` | `occupation-recommend/.../service/impl/SemanticMatchServiceImpl.java` |
| VO | `SemanticMatchVO.java` | `occupation-recommend/.../vo/SemanticMatchVO.java` |
| DTO | `SemanticMatchDTO.java` | `occupation-recommend/.../dto/SemanticMatchDTO.java` |

**核心逻辑**：调用 DeepSeek 做 JD 文本与简历文本的语义相似度计算，结果缓存 Redis 24h。

#### 匹配分数也在以下功能中体现：

- **`HrResumeAiServiceImpl.java`** — `screen()` 方法返回 `matchScore`（0-100），`rankByMatch()` 批量排序
- **`ResumeAiServiceImpl.java`** — `review()` 方法返回 `score`（0-100），简历综合评分
- **`JobMatchServiceImpl.java`** — 推荐系统的匹配计算

## 关键架构说明

- **所有 AI 调用统一走 `AiChatClient`**，不直接调 HTTP
- **AI 不可用时自动降级**：简历诊断降级为规则打分，面试问题降级为模板问题
- **`app.ai.enabled` 是总开关**，关闭后所有功能仍可用（降级模式）
- **API Key 通过环境变量或 `application-local.yml` 注入**，不写进仓库
- **前端超时统一设为 90s**（大模型首字延迟 + 长回复需要）
